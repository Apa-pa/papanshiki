# がくしゅうラボ 仕様メモ

## 目的

こどもごとの学習状況を、カテゴリ別の習熟度として確認できるページを作成する。

さんすうであれば、たしざんをどれくらい理解できていて、どれくらいのスピードで解けるのかを 100 点満点で表示する。足し算だけでなく、ひきざん、かけざん、わりざん、分数、小数、時計などへ段階的に拡張できる設計にする。

初期実装では、さんすうとこくごの 2 科目を用意し、それぞれ 3 カテゴリ程度から開始する。

## 基本フロー

1. がくしゅうラボを開く
2. ユーザーを選ぶ
3. 科目を選ぶ
4. カテゴリを選ぶ
5. 20 問テストを受ける
6. 結果を保存する
7. 同じユーザーに、テスト点数と同じポイントを付与する

## 初期カテゴリ

### さんすう

- たしざん
- ひきざん
- かけざん

### こくご

- ひらがな・カタカナ
- ことばの意味
- かんじ 1 年生

## テスト仕様

- 1 カテゴリにつき 20 問出題する。
- 1 問ずつ表示する。
- 20 問すべて終了したときだけ結果を保存する。
- 途中離脱した場合は、習熟度もポイントも更新しない。
- テスト中は現在の進捗を表示する。
  - 例: `7 / 20`
  - 進捗バー
  - 現在の正答数
- 習熟度スコアは最後の結果画面で表示する。

## 習熟度スコア

習熟度は 0 から 100 点で表示する。

初期案:

```text
習熟度 = 正答率点 75 点 + スピード点 25 点
```

正答率を重めにしつつ、解答スピードも反映する。

カテゴリごとに標準解答時間を持たせる。

例:

- たしざん: 1 問あたり 5 秒
- ひきざん: 1 問あたり 6 秒
- かけざん: 1 問あたり 7 秒
- こくご系: 1 問あたり 8 秒から 10 秒程度

結果表示例:

```text
たしざん
正答数: 18 / 20
平均時間: 4.6 秒
習熟度: 86 点
評価: とくい
```

## 評価ラベル

習熟度に応じて、こどもにも分かりやすい評価ラベルを表示する。

- 0 から 39: れんしゅう中
- 40 から 69: いいかんじ
- 70 から 89: とくい
- 90 から 100: マスター

## ポイント付与

- テスト完了時、習熟度スコアと同じポイントを付与する。
- 例:
  - 習熟度 42 点なら 42 pt
  - 習熟度 86 点なら 86 pt
  - 習熟度 100 点なら 100 pt
- ポイントは、最初に選択したユーザーに付与する。
- テスト後に別ユーザーを選び直す仕様にはしない。
- 途中離脱時はポイントを付与しない。

既存仕様との接続方針:

- 既存の `ranking.js` のポイント保存仕様に合わせる。
- 可能であれば既存の共通関数を利用する。
- 既存関数がユーザー選択ダイアログ前提の場合は、選択済みユーザーに直接付与できる実装を検討する。
- プレイ履歴用の `gameId` はカテゴリ別にする。

例:

```js
learning_labo_math_addition
learning_labo_math_subtraction
learning_labo_math_multiplication
learning_labo_japanese_kana
learning_labo_japanese_words
learning_labo_japanese_kanji_grade1
```

## 保存データ

localStorage キー:

```text
papan_learning_labo_records_v1
```

最後に選択したユーザー:

```text
papan_learning_labo_last_user
```

保存データはユーザー別、カテゴリ別に持つ。

例:

```js
{
  "たろう": {
    "math_addition": {
      "mastery": 86,
      "bestMastery": 91,
      "attempts": 3,
      "lastCorrect": 18,
      "totalQuestions": 20,
      "lastAccuracy": 0.9,
      "lastAverageSeconds": 4.6,
      "lastTotalSeconds": 92,
      "lastTestedAt": "2026-04-29T12:00:00.000Z"
    }
  }
}
```

## 画面に表示する情報

カテゴリ一覧では、選択中ユーザーの習熟度を表示する。

表示項目:

- 習熟度スコア
- 評価ラベル
- 前回の正答数
- 前回の平均解答時間
- 受験回数
- 最終受験日

未受験の場合は、まだテストを受けていない状態として表示する。

## 各ファイルの役割

### `labo.html`

がくしゅうラボの画面本体。

ユーザー選択、科目選択、カテゴリ選択、テスト画面、結果画面を表示する。

### `style.css`

がくしゅうラボ共通デザイン。

既存のぱぱん式に合わせて、タッチしやすく、やさしい色合いの UI にする。

### `main.js`

共通ロジック。

- 画面遷移
- ユーザー選択
- カテゴリ一覧表示
- テスト進行
- 採点
- 習熟度計算
- localStorage 保存
- ポイント付与

### `categories.js`

科目とカテゴリの定義。

- 科目 ID
- 科目名
- カテゴリ ID
- カテゴリ名
- 問題数
- 標準解答時間
- 読み込むテスト JS

### `tests/*.js`

カテゴリごとの問題生成、または問題データを持つ。

さんすうは自動生成を基本にする。

こくごは問題プールから 20 問を抽出する形を基本にする。

## カテゴリ追加の方針

新しいカテゴリを追加するときは、基本的に以下の 2 ステップで済むようにする。

1. `tests/` にカテゴリ用 JS を追加する。
2. `categories.js` にカテゴリ定義を追加する。

## 実装時の追加作業

- `index.html` に「がくしゅうラボ」への入口を追加する。
- `service-worker.js` のキャッシュ対象に、がくしゅうラボ関連ファイルを追加する。
- 必要に応じて `dashboard.html` に学習ラボの導線や集計表示を追加する。
- ポイント付与とプレイ履歴保存は、既存の `ranking.js` の仕様に合わせて実装する。
