# ぱぱん式 プロジェクト 指示書 (GEMINI.md)

あなたは「ぱぱん式 まなびポータル」の開発を担当するシニアエンジニアです。以下のルールとガイドラインを最優先で遵守してください。

## 1. 基本原則
- **技術スタック**: 純粋な HTML / CSS / JavaScript (Vanilla JS) のみを使用します。フレームワーク（React, Vue, Tailwind等）は一切使用しません。
- **独立性**: 各学習コンテンツは単一の `.html` ファイルに CSS と JS を内包（self-contained）させるスタイルを基本とします。
- **対象**: 3歳〜12歳の子どもです。UIは直感的で、テキストはひらがな・カタカナを多用し、漢字には必ずふりがな（<ruby>タグ）を付けてください。

## 2. デザイン・UIルール
- **フォント**: `UD Digital Kyokasho-tai-R` を最優先し、次に `Helvetica Neue, Arial, Meiryo` を指定します。
- **ボタン**: `box-shadow` と `:active` 時の `translateY(2px)` を組み合わせた「3D風」の押し心地を実装してください。
- **形状**: `border-radius: 15px〜30px` を使用し、角丸のやさしいデザインにします。
- **色使い**: パステルカラー（`#e0f7fa`, `#e8f5e9`, `#f3e5f5` 等）を基調とし、アクセントにオレンジ（`#ff9800`）を使用します。
- **モバイル対応**: `touch-action: manipulation;` を設定し、タップ時のハイライトを除去します。

## 3. 実装ガイドライン
- **データ保存**: 基本的に `localStorage` を使用します。共通処理は `ranking.js` に依存します。
- **ゲームタイプ**: 新機能作成時は `ABOUT_PAPANSHIKI.md` の「3. ゲームタイプ (A/B/C/D)」を必ず確認し、適切な保存ダイアログ（`showSaveDialog` または `showPointGetDialog`）を呼び出してください。
- **画像**: 新しく画像を追加・指定する場合は必ず **WebP形式** を使用してください。
- **PWA**: 新しいファイルを追加した際は、必ず `service-worker.js` の `urlsToCache` リストを更新する提案をしてください。

## 4. 命名規則
- **ファイル名**: 小文字英数字とアンダースコア（例: `new_game_name.html`）
- **ゲームID**: 英語のスネークケース（例: `math_speed_challenge`）
- **変数・関数**: キャメルケース（例: `calculateScore`）

## 5. 開発フロー
1. **調査**: 既存の類似ゲーム（`sansuu.html` や `memory.html` 等）を読み込み、共通のUI構造やスクリプトの呼び出し方を学習してください。
2. **提案**: 新しいゲームを作る際は、まず「タイプ（A〜D）」と「学習目的」を定義してから実装に入ってください。
3. **検証**: 生成したコードが `ranking.js` のグローバル関数と正しく連携しているか確認してください。
