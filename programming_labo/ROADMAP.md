# プログラミングラボ アップデート計画

## 目的

プログラミングラボは、ぱぱん式のキャラクターや素材に命令をつけて動かす、子ども向けのビジュアルプログラミング教材です。

今後は、自由に動く作品を作るだけでなく、簡単なミニゲームを作れる機能と、お題に沿ってゲームを完成させるクイズモードを追加していきます。

## 現在の初版機能

- 素材を選んでキャンバスに配置できる
- 複数のキャラを配置できる
- 配置したキャラを選択できる
- キャラごとに複数の命令をつけられる
- 「上からじゅんに」実行される命令を使える
- 「ずっと」動く命令を使える
- 「もしも」ルールを使える
- 再生、とめる、さいしょにもどす、ぜんぶけすができる
- ポイント付与、保護者おすすめ、すごろくミッションに対応している

## 次期アップデートの柱

1. ミニゲームを作りやすくするコマンド追加
2. キャラの見た目やサイズを編集できる機能
3. お題に沿ってプレイヤーがゲームを作るクイズモード
4. お題クリア判定と報酬・履歴の仕組み

## 追加コマンド案

- スタート位置にもどる
- 速くする
- ゆっくりにする

### ずっと命令

- ずっと追いかける
- ずっとにげる
- ずっと上下に動く
- ずっと左右に動く
- ずっとふわふわ動く

### もしもルール

- ぶつかったら 花丸を表示する
- ぶつかったら うごきだす
- ぶつかったら とまる
- ぶつかったら きえる
- ぶつかったら スタートにもどる
- ぶつかったら ポイントが入る
- ぶつかったら セリフを出す
- ぶつかったら 音を鳴らす
- どんぐりにふれたら どんぐりがきえる
- ハートにふれたら クリア

## キャラ編集機能

### 初期サイズ変更

選択中のキャラに対して、再生前の大きさを変更できるようにします。

候補UI:

- 大きさスライダー
- 小・中・大ボタン
- 選択中キャラのプレビュー

### 命令としてのサイズ変更

既存の「大きくなる」「小さくなる」は、再生中に変化する命令として残します。

区別:

- 編集機能: 最初から大きさを変える
- 命令機能: 再生中に大きさが変わる

## クイズモード案

クイズモードでは、プレイヤーが指定された機能をもつ簡単なゲームを作ります。

基本の流れ:

1. お題を読む
2. あらかじめ素材が置かれている
3. プレイヤーが命令やルールをつける
4. 再生する
5. 条件を満たしたらクリア
6. ポイント、スタンプ、すごろくミッションなどに連携する

自由制作モードとは別に、段階的にプログラミングの考え方を学べるモードとして用意します。

## クイズお題案

### レベル1: 動かす

- アンをハートまで進めよう
- ピッピをどんぐりまで進めよう
- ヒーを右のゴールまで進めよう

### レベル2: ぶつかったら

- アンがヒーにぶつかったら花丸が表示されるかけっこゲーム
- ピッピがモンスターにぶつかったらモンスターがきえるゲーム
- キャラがどんぐりを拾うと、どんぐりがきえるゲーム

### レベル3: ずっと命令

- アンを右に進み続けさせて、はしで跳ね返らせよう
- モンスターを上下に動かそう
- ピッピが動くどんぐりを追いかけるゲームを作ろう

### レベル4: 順番と連動

- アンとヒーとピッピが順番にうごいてハートがいるところまで進むリレーゲーム
- アンがどんぐりを拾ったら、ヒーがうごきだすゲーム
- ヒーがハートに着いたら、ピッピが花丸を出すゲーム

## クリア判定の考え方

クイズモードでは、プレイヤーがどの命令カードを置いたかではなく、再生中に実際にお題どおりの結果が起きたかを判定します。

例:

```js
{
    id: "quiz_hit_hanamaru",
    title: "アンがヒーにぶつかったら花丸",
    clearConditions: [
        { type: "objectHit", a: "an", b: "hi" },
        { type: "effectShown", effect: "hanamaru" }
    ]
}
```

この方式にすると、子どもが自分なりの解き方を試せます。

## 実装優先度

1. キャラサイズ変更（実装済み）
2. ぶつかったら系コマンド追加（花丸、うごきだす、とまる、きえる、スタートにもどるは実装済み。ポイント、セリフ、音は後回し）
3. 花丸、セリフ、音などの演出（花丸のみ実装済み。セリフ・音は後回し）
4. クイズモードの画面とデータ構造（サンプル1問の土台を実装済み）
5. クイズお題を10問程度追加
6. クイズクリア履歴と報酬
7. 作品保存、再編集

## 実装メモ

- コマンドは現在の `commands`, `loops`, `rules` の3分類を維持する
- クイズモードでは初期配置データとクリア条件データを別ファイル化する
- クリア判定はイベントログ方式にすると拡張しやすい
- 「花丸を表示する」などの演出は、キャラではなく一時的なエフェクトとして管理する
- 「ぶつかったら うごきだす」は、停止中のキャラに `loops` を有効化する形で実装できる
- 「ぶつかったら」系ルールが複数ある場合は、ぶつかるたびに上から1つずつ実行し、最後まで進んだら一番上に戻る
- お題クリア時は `showPointGetDialog` と `addMissionStock` に連携する
