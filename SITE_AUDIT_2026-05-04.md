# ぱぱん式 サイト改善監査メモ

作成日: 2026-05-04

## 対象

- プロジェクト全体の HTML / CSS / JS / 画像 / PWA 設定
- 特に入口導線となる `index.html` `index2.html`、共通基盤 `ranking.js` `service-worker.js`、回遊導線の強い `dashboard.html` `blog/` `learning_labo/` `performance_labo/` を重点確認

## 先に結論

- いちばん強い資産は「コンテンツ量」と「学習後の報酬導線」です。遊ぶ理由は十分あります。
- いちばん大きい課題は「入口の整理不足」と「SEO 基本設定不足」です。
- 次点の課題は「トップ周辺の重複」「Service Worker の手運用負荷」「新規追加コンテンツのリンク・キャッシュ追従漏れ」です。
- 改修は、まず `index.html` を学習ポータルのハブとして再設計し、そのあと SEO と共通化を進めるのが最も費用対効果が高いです。

---

## 1. ユーザー導線

### 現状の良い点

- `index.html` から主要コンテンツへの入口はかなり豊富。
- 学習後の報酬導線が明確。
  - `record.html`
  - `gacha.html`
  - `bank.html`
  - `stock.html`
  - `room.html`
  - `sugoroku/sugoroku.html`
- `dashboard.html` には保護者向けの「おすすめコンテンツ」設定があり、継続利用の仕組みとして強い。

### 課題

- トップページが情報過多。
  - `index.html` は約 78KB の単一 HTML で、カテゴリ・カード数が多く、初見ユーザーが「何から始めればよいか」判断しづらい。
- `index.html` と `index2.html` の役割分担が曖昧。
  - 見た目も構造もかなり近く、入口が二重化している。
- 年齢・目的別の最短導線が弱い。
  - 例: 「3-5歳向け」「1日10分だけ」「算数を強化したい」「親向け確認」などの導線がトップで弱い。
- 多くの個別ゲームが「ホームへ戻る」だけで終わる。
  - クリア後の次アクションが「同ジャンルの次教材」「記録を見る」「今日のおすすめへ」が少ない。
- `index2.html` にリンク不整合あり。
  - `job_learn.html` へのリンクが `jobs/job_learn.html` ではなく、現状では切れている。

### 改善案

- トップのファーストビュー直下に 4 導線を固定配置する。
  - はじめてあそぶ
  - 年齢でえらぶ
  - 今日のおすすめ
  - おうちのかたへ
- カテゴリ一覧の前に「おすすめ3件」を出す。
  - 日替わりミッション
  - 前回の続き
  - 保護者おすすめ
- 各ゲーム終了画面に次導線を追加する。
  - 同カテゴリの次の1本
  - 関連ラボ
  - 記録ページ
  - すごろく/ガチャ
- `index.html` を主ハブに統一し、`index2.html` は廃止または明確な特集ページ化を検討。

## 2. SEO

### 現状

- HTML 総数: 137
- `meta description` あり: 19
- `meta description` なし: 118
- `canonical` あり: 0
- `h1` あり: 106
- `h1` なし: 31
- `index.html` と `index2.html` は OGP はあるが、`meta description` がない。
- `index.html` `index2.html` には `h1` が見当たらない。
- ブログ配下は比較的 SEO 基本設定が入っている。
- 多くのゲームページは `<title>` のみ、またはタイトル品質にばらつきがある。

### 課題

- 検索結果での説明文不足。
- 類似ページが多く、検索エンジンにページ意図が伝わりにくい。
- `canonical` がないため、類似ページや一覧ページの評価が分散しやすい。
- 学習コンテンツの検索流入を拾える設計になりきっていない。
  - 例: 「小1 漢字 ゲーム」「分数 練習 無料」「時計 読み方 ゲーム」など

### 改善案

- まずトップ・カテゴリハブ・主要集客ページから `meta description` を整備。
- `index.html` に 1 つだけ明確な `h1` を設置。
- 各主要ページに `canonical` を追加。
- カテゴリ別ハブを SEO 用に作る。
  - さんすう
  - こくご
  - 英語
  - 脳トレ
  - お金
  - 理科
- ブログ記事から該当学習ページへの文脈リンクを増やす。
- 将来的に `sitemap.xml` と `robots.txt` も追加対象。

## 3. 内部リンク

### 現状の良い点

- ブログ記事には一部、本編へのリンクがある。
  - `blog/article002.html` -> `bank.html` `stock.html`
  - `blog/article008.html` -> `dashboard.html`
- `bank.html` と `stock.html` の相互リンクは良い。
- `room.html` 周辺はショップ・一覧との往復ができる。

### 課題

- コンテンツ間の横移動が弱い。
  - 多くのページで戻り先が `index.html` のみ。
- トップから深いページへは行けるが、深いページから別学習への回遊が少ない。
- ブログ一覧 `blog/index.html` から本編カテゴリへの導線が弱い。
- `index2.html` の `job_learn.html` リンク切れは内部リンク品質を落とす。
- `communication/` `jobs/` `learning_labo/` `performance_labo/` など新しい塊が、トップ以外の主要ページから十分に再送客されていない。

### 改善案

- 全ゲーム共通で「関連コンテンツ」導線を追加する。
  - 同年齢帯
  - 同教科
  - 難易度アップ版
- ブログ末尾に「この記事で紹介した学習はこちら」を固定化。
- `record.html` `dashboard.html` `calendar.html` から直接プレイ再開リンクを出す。
- カテゴリハブから各ゲームへのリンクだけでなく、ゲームページからカテゴリハブへ戻す。

## 4. 表示速度

### 現状の観測

- 大きい画像が複数残っている。
  - `sugoroku/background.png` 約 9.0MB
  - `header02.png` 約 5.6MB
  - `header2.png` 約 2.0MB
  - アバター PNG 群も大きいものがある
- WebP 化は進んでいるが、PNG 版が多数残存。
- `index.html` `index2.html` は CSS/JS 内包量が大きく、ページ自体が重い。
- ほぼ全ページで `ranking.js` を読み込む。
  - `ranking.js` 約 54KB
- 外部 CDN/外部 API 依存あり。
  - `Chart.js`
  - `Matter.js`
  - `https://api.exchangerate-api.com/...`
- `service-worker.js` は手動の `urlsToCache` 配列で運用。
- Service Worker キャッシュ漏れがある。
  - 例: `index2.html`
  - `communication/`
  - `jobs/`
  - `learning_labo/labo.html`
  - `performance_labo/*`
  - `sugoroku/sugoroku.html`
  - `sansuu_bunsu.html`
  - 一部ブログ

### 課題

- 新規ページ追加時に SW 更新漏れが起きやすい。
- 巨大画像の存在が、保守上も転送量上も不利。
- 単一巨大トップのため、初回表示負荷が高くなりやすい。
- CDN や外部 API が遅いと、一部機能体験が不安定になる。

### 改善案

- 使っていない PNG 原本を整理し、公開配信は WebP 優先に統一。
- `sugoroku/background.png` などの巨大原本は公開配下から外す検討。
- Service Worker を「手書き全列挙」から段階的に改善。
  - 最低限: 新規ページのチェックリスト自動化
  - 理想: 生成スクリプト化
- トップの画像・セクションを遅延読込寄りに整理。
- 外部ライブラリは必要ページだけに限定できているが、将来的にはローカル同梱も検討余地あり。

## 5. モバイル UI

### 現状の良い点

- 多くのページで子ども向けに大きめ UI。
- `touch-action` や押下感のあるボタン設計が全体に浸透。
- `communication/hub.html` `learning_labo/labo.html` `performance_labo/labo.html` は比較的モバイル志向が明確。

### 課題

- トップのカード量が多く、モバイルでは縦長になりすぎる。
- `index.html` の初見時に「スクロール疲れ」が起きやすい。
- ページごとに戻るボタン・終了ボタンの位置や文言が揺れている。
- 一部ページは固定サイズ演出が多く、小画面で圧迫しやすい。
- `maximum-scale=1.0, user-scalable=no` を使っているページがあり、アクセシビリティ上は慎重に扱いたい。
  - 例: `communication/hub.html`

### 改善案

- トップは「おすすめ」「年齢」「教科」で一段階絞ってからカードを見せる。
- 共通のモバイルヘッダー/戻る導線を用意する。
- プレイ画面以外は基本的にズーム禁止を避ける。
- 主要ページから順に、360px 幅前提の確認リストを作る。

## 6. 学習ポータルとしての回遊性

### 現状の良い点

- ポイント、どんぐり、ガチャ、部屋、株、すごろくが回遊エンジンとして機能している。
- `dashboard.html` のおすすめ設定は、親主導の回遊導線としてかなり良い。
- `learning_labo` と `performance_labo` は「まとめて体験する」入口として強い。

### 課題

- 学習コンテンツ同士のレベル設計が画面上で見えにくい。
  - 初級 -> 中級 -> 応用 の並びが弱い
- 「今日やること」がトップと各ゲーム終了画面でつながっていない。
- ブログ、ダッシュボード、記録、カレンダーが個別に良いのに、相互送客が弱い。
- `learning_labo` のような新しい学習ハブが、トップ以外から再訪されにくい。

### 改善案

- 「学習ルート」を作る。
  - 例: 小1 ことば入門ルート
  - 例: かけ算強化ルート
  - 例: 集中力チャレンジルート
- `dashboard.html` に「つぎにおすすめの3本」を出す。
- `record.html` に未達成目標から直接飛べる導線を追加。
- ブログ記事の中に「今すぐ試す」ブロックを標準化。
- ラボ系の結果ページから、弱点カテゴリの通常ゲームへ流す。

## 7. 重複コード・不要ファイル

### 確認できた重複/整理候補

- `index.html` と `index2.html`
  - 役割がかなり近い。片方を統合または縮小候補。
- `rail.html` と `rail2.html`
  - 差分はあるが近縁ページ。用途を整理したい。
- `train.png`
  - `performance_labo/train.png` と同一ハッシュ。
- 巨大 PNG 原本群
  - `header01.png`
  - `header02.png`
  - `header2.png`
  - `gacha_machine.png`
  - 複数アバター PNG
- ブログの雛形/検証ページ
  - `blog/template.html`
  - `blog/test-article.html`
  - 本番公開対象なら noindex 相当の扱い、または公開配下から外したい。
- `header01.clip`
  - 配信用ではなく制作素材なら公開配下から外したい。
- プロトタイプページ
  - `fly_catch_prototype.html`
  - `volleyball_rally_prototype.html`
  - 本番導線に載せないなら整理候補。

### 保守上の構造課題

- 各ページにスタイルと UI パターンが内包され、共通化率が低い。
- `ranking.js` が多機能化しており、今後さらに肥大化しやすい。
- Service Worker のキャッシュ対象が手運用で、更新漏れを誘発しやすい。

### 改善案

- 第1段階は「不要ファイル整理」と「公開配下素材の棚卸し」。
- 第2段階で共通 UI を小さく切り出す。
  - 共通ヘッダー
  - 共通戻るボタン
  - 共通 CTA
  - 共通カード
- `ranking.js` は将来的に責務分割を検討。
  - 記録
  - 通貨/報酬
  - 目標/ミッション
  - ダイアログ

## ページ別メモ

### `index.html`

- 現在の中心導線として最重要。
- OGP はあるが `description` と `h1` が不足。
- 情報量が多く、初回体験は強いが迷いやすい。

### `index2.html`

- `index.html` とかなり近い。
- `job_learn.html` へのリンク誤りあり。
- 統合候補として優先的に整理したい。

### `dashboard.html`

- 回遊性向上の起点として非常に有望。
- ここから通常学習ページへ再送客する UI を増やすと強い。

### `service-worker.js`

- ネットワーク優先の方針自体は理解しやすい。
- ただし手作業の `urlsToCache` は運用負荷が高い。

### `blog/`

- SEO の受け皿として良い。
- ただし本編への内部リンク強化余地がまだ大きい。

### `learning_labo/` `performance_labo/`

- 学習ポータルの中核になれるまとまり。
- トップ以外からの再訪導線強化で価値がさらに上がる。
